## Scope

This phase has two distinct UI surfaces:

1. **LIFF Registration Page** — A single-page form at `/liff/events/[id]/register` inside LINE's mobile webview. Used by both members and admins.
2. **LINE Flex Message Card (Repost)** — Flex Message bubbles posted to the LINE group chat in response to registration changes. Rendered by LINE's native card renderer.

Design contracts for both surfaces are declared below.

---

## Visuals

**Primary focal point:** The "ลงทะเบียน" (Register) or "ยกเลิก" (Cancel) button at the top of the member list — it is the primary CTA and anchors user action on the page.

**Secondary focal point:** The numbered member list below the button — displays who is registered and provides context for registration state.

**Admin UI:** Inline (X) remove icons next to each member name (visible to admins only) and a "ปิดรับลงทะเบียน" close button at the bottom of the page (admin-only).

**State indicators:** Full event shows a red/orange badge at the top with "เต็มแล้ว" text; closed event disables the register button with "ปิดรับลงทะเบียนแล้ว" label.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between member name and remove icon |

| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |

Exceptions:
- Primary button (Register/Cancel) minimum height: 44px (touch target requirement) — matches established pattern from ProfileForm

- Remove icon button padding: 8px (touch target compatibility)

Source: Established patterns from `apps/web/app/liff/profile/page.tsx` and Phase 4 `EventCreate` page.

### LIFF Registration Page Layout Pattern

```
<div className="max-w-sm mx-auto px-4 py-8">
  <h1 className="text-2xl font-bold mb-6">ลงทะเบียน</h1>
  
  {/* Full state badge — if registeredCount === maxPlayers */}
  {isFull && (
    <Badge className="mb-4 bg-destructive text-destructive-foreground">
      เต็มแล้ว
    </Badge>
  )}
  
  <Card className="mb-6">
    <CardContent className="pt-6 space-y-4">
      {/* Event info section */}
      <div className="space-y-1">
        <p className="text-sm font-bold">{event.title}</p>
        <p className="text-sm text-muted-foreground">{formatDate(event.eventDate)}</p>
        <p className="text-sm">{event.venueName}</p>
      </div>
      
      {/* Count display */}
      <div className="text-sm">
        <span className={isFull ? "text-destructive" : "text-foreground"}>
          {registeredCount}/{maxPlayers} คน
        </span>
      </div>
    </CardContent>
  </Card>
  
  {/* Register/Cancel button — primary CTA */}
  <Button
    className="w-full min-h-[44px] mb-6"
    disabled={isFull || isClosed}
    onClick={handleRegisterToggle}
  >
    {isCurrentMemberRegistered ? "ยกเลิก" : "ลงทะเบียน"}
  </Button>
  
  {/* Member list */}
  <Card>
    <CardContent className="pt-6">
      <p className="text-sm font-bold mb-4">ลงทะเบียนแล้ว</p>
      <ul className="space-y-3">
        {registrations.map((reg, idx) => (
          <li key={reg.id} className="flex items-center justify-between text-sm">
            <span>{idx + 1}. {reg.displayName}</span>
            {isAdmin && (
              <button
                className="text-muted-foreground hover:text-destructive"
                onClick={() => handleRemoveMember(reg.id)}
                aria-label={`Remove ${reg.displayName}`}
              >
                <X className="h-4 w-4" />
              </button>
            )}
          </li>
        ))}
      </ul>
    </CardContent>
  </Card>
  
  {/* Admin close/reopen button — bottom of page */}
  {isAdmin && (
    <Button
      variant="outline"
      className="w-full min-h-[44px] mt-6"
      onClick={handleToggleClose}
    >
      {isClosed ? "เปิดรับลงทะเบียน" : "ปิดรับลงทะเบียน"}
    </Button>
  )}
</div>
```

Source: Established pattern from Phase 4 event creation page.

## Interaction States

### LIFF Registration Page States

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading (fetching event data) | On mount, before fetch resolves | Full-screen centered `<Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />` |
| Page ready | Event data loaded | Render full page with event info, register button, member list |
| Registering (POST in-flight) | User taps "ลงทะเบียน" button | Register button disabled + spinner, list refresh disabled |
| Registered (member added to list) | POST succeeds | Button toggles to "ยกเลิก", member name appears in numbered list |
| Cancelling (DELETE in-flight) | User taps "ยกเลิก" button | Cancel button disabled + spinner |
| Cancelled (member removed from list) | DELETE succeeds | Button toggles to "ลงทะเบียน", member name disappears from list |
| Full state | `registeredCount === maxPlayers` | Red "เต็มแล้ว" badge at top, register button disabled with label "เต็มแล้ว", member list still visible |
| Closed state | `event.status === "closed"` | Register button disabled with label "ปิดรับลงทะเบียนแล้ว", member list still visible |
| Page focus refresh | User returns from LINE chat (visibilitychange event) | Silently fetch updated registration list and re-render without spinner (per D-04) |
| Admin removing member | Admin taps (X) icon next to member | Inline spinner on remove button, member row disabled temporarily |
| Member removed | Admin removal succeeds | Member row disappears from list, count updates |
| Admin closing registration | Admin taps "ปิดรับลงทะเบียน" button | Button disabled, calls PATCH status endpoint |
| Registration closed | PATCH succeeds | Button re-enables with label "เปิดรับลงทะเบียน", register button now shows "ปิดรับลงทะเบียนแล้ว" |
| Admin reopening | Admin taps "เปิดรับลงทะเบียน" button | Calls PATCH status endpoint to set status back to "open" |
| Error state (registration conflict) | POST returns 409 (already registered or full) | Toast error with Thai message, form stays open |
| Error state (closed event) | POST returns 409 (event closed) | Toast error "ปิดรับลงทะเบียนแล้ว", form stays open |
| Error state (generic) | API returns 5xx or network error | Toast error "ลงทะเบียนไม่สำเร็จ กรุณาลองใหม่", form stays open |
| Auth gate (not logged in) | LIFF session missing or invalid | Redirect to login or show error message (handled by LiffProvider) |

Source: Established from Phase 4 + CONTEXT.md D-01 through D-17, RESEARCH.md pitfalls.

### Flex Message Card Repost States (LINE Group Chat)

Per CONTEXT.md D-05 through D-09, D-12, D-17:

| State | Trigger | Alt Text | Count Display | Notes |
|-------|---------|----------|---------------|-------|
| Member registered | POST succeeds | `[MemberName] ลงทะเบียนแล้ว ({count}/{max} คน)` | Updated count in green if open, red if full | Card reposts with notification style |
| Member cancelled | DELETE succeeds | `[MemberName] ยกเลิกแล้ว ({count}/{max} คน)` | Updated count | Card reposts with notification style |
| Admin removed member | Admin DELETE succeeds | No member name shown (privacy) — count only: `({count}/{max} คน)` | Updated count | Card reposts without naming removed member |
| Registration closed | PATCH status=closed | `ปิดรับลงทะเบียนแล้ว ({count}/{max} คน)` | Final count | Card reposts with closed state notification |
| Registration reopened | PATCH status=open | Not explicitly specified — no repost on reopen? Or `เปิดรับลงทะเบียน`? | Updated count | [CLARIFICATION NEEDED: Does reopen trigger repost?] |

Per D-08: After each mutation, `events.lineMessageId` is updated to point to the latest card.
Per D-07: Old cards remain in chat as history.

Source: CONTEXT.md D-05 through D-17, RESEARCH.md Pattern 4 (Repost Card).

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Touch targets minimum 44px | Register/Cancel button: `min-h-[44px]`; Admin close button: `min-h-[44px]`; Remove icon button padding ensures clickable area |
| Member list readability | Numbered list with clear spacing (space-y-3); member names at 16px body text |
| Form labels | All sections have heading text ("ลงทะเบียนแล้ว", "ปิดรับลงทะเบียน") — not requiring `<Label>` component since no `<Input>` elements |
| Error messages | Toast notifications for user feedback; visually distinct destructive red for error states |
| Loading state | `<Loader2 aria-label="Loading">` with `text-muted-foreground` spinner — visible on full-page and button-level loading |
| Disabled state | Register button disabled during submit; close button disabled during admin action; visual feedback via `disabled` attribute |
| Remove icon accessibility | Inline (X) button has `aria-label={`Remove ${displayName}`}` for screen reader users |
| Focus management | No custom focus management required — browser native focus follows interactive elements |

Source: Established from Phase 4 accessibility patterns and WCAG 2.1 AA compliance via shadcn/ui components.
