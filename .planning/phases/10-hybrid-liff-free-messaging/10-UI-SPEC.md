## Interaction States

Phase 10 adds two new interaction surfaces that require defined states:

### External Browser Auth Redirect (new in this phase)

| State | UI | Copy |
|-------|----|------|
| Initiating LINE Login redirect | Full-screen centered `Loader2` spinner (h-8 w-8, muted-foreground) | None — redirect happens immediately |
| After OAuth return, re-initializing | Same full-screen spinner | None |
| Auth error (token not received) | Inline error message centered on screen | See Copywriting Contract below |

### sendMessages After Register / Cancel (new in this phase)

| State | UI | Duration |
|-------|----|----------|
| API call in-flight | Button shows `Loader2` spinner, disabled | Until API responds |
| sendMessages succeeds | Existing success toast fires | 3s auto-dismiss |
| sendMessages fails silently | No visible UI change — error logged to console only; server already updated group via pushMessage fallback | n/a |

## Page Layout Contract

All moved pages retain existing layout structure. No visual redesign — this is a structural refactor.

| Page | New Path | Layout Note |
|------|----------|-------------|
| Event details / register | `/events/[id]` | No `<Nav />` — use route group `(liff)` or suppress Nav on these pages |
| Profile setup | `/setup` | No `<Nav />` |
| Profile edit | `/profile` | No `<Nav />` |
| Create event | `/events/create` | No `<Nav />` |
| Event templates | `/events/templates` | No `<Nav />` |
| Template edit | `/events/templates/[id]/edit` | No `<Nav />` |

Note from RESEARCH.md open question: Nav should NOT render on LIFF pages. Use a `(liff)` route group with its own layout (no Nav) and place LiffProvider in root layout per D-11. This resolves the nav bar question without conditional logic.
