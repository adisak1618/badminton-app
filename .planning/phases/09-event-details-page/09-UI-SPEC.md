## Page Layout Contract

### `/liff/events/[id]` (unified event details + register page)

Layout: Single-column, `max-w-sm mx-auto px-4 py-8`. Mobile-first, no desktop breakpoints needed (LIFF context).

**Section order (top to bottom):**

1. **Page heading** — `text-2xl font-bold mb-6` — text: "ลงทะเบียน"
2. **Full state badge** — shown only when `isFull && !isClosed` — `inline-block mb-4 bg-destructive text-destructive-foreground px-2 py-1 rounded text-sm` — text: "เต็มแล้ว"
3. **Cancelled state badge** — shown only when `event.status === "cancelled"` — same badge style as full badge — text: "ยกเลิกแล้ว"
4. **Event info card** — `Card` with `CardContent pt-6 space-y-1`:
   - Title: `text-sm font-bold`
   - Date/time (th-TH locale): `text-sm text-muted-foreground`
   - Venue name (if present): `text-sm`
   - Venue maps link (if present): `text-sm text-primary underline` linking to `venueMapsUrl`
   - Count: `text-sm` — `{registeredCount}/{maxPlayers} คน` — color `text-destructive` when full, `text-foreground` otherwise
5. **Register/Cancel button** — `Button w-full min-h-[44px] mb-6` — state-dependent label (see Copywriting Contract)
6. **Registration list card** — `Card` with `CardContent pt-6`:
   - Section label: `text-sm font-bold mb-4` — text: "ลงทะเบียนแล้ว"
   - Empty state: `text-sm text-muted-foreground`
   - Registered member row: `flex items-center justify-between text-sm` — `{idx+1}. {displayName}` with admin X button on right
7. **Admin close/reopen button** — shown only when `isAdmin === true` — `Button variant="outline" w-full min-h-[44px] mt-6`

### Loading state
Full-screen centered spinner: `flex items-center justify-center min-h-screen` with `Loader2 h-8 w-8 animate-spin text-muted-foreground`.

### Error state
Container `max-w-sm mx-auto px-4 py-8` with `<p className="text-destructive">` showing error message.

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Page mount, LIFF not ready | Full-screen spinner |
| Registered | `currentMemberRegistrationId !== null` | Button shows "ยกเลิก" |
| Full (not registered) | `registeredCount >= maxPlayers && !isRegistered` | Button disabled, shows "เต็มแล้ว", full badge visible, count text red |
| Closed | `event.status === "closed"` | Button disabled, shows "ปิดรับลงทะเบียนแล้ว" |
| Cancelled | `event.status === "cancelled"` | Button disabled, shows "ยกเลิกแล้ว", cancelled badge visible |
| Submitting | API call in flight | Button disabled with inline Loader2 spinner + in-progress label |
| Admin | `isAdmin === true` | X remove button visible on each member row; admin close/reopen button appears below list |

## Flex Message Card Contract

**Single CTA button rule (D-04 from CONTEXT.md):**

| Event state | Button label | URL |
|-------------|--------------|-----|
| Open with available slots | ลงทะเบียน | `/liff/events/[id]` |
| Full or closed or cancelled | รายละเอียด | `/liff/events/[id]` |

**Notification line in repost card body (D-06, D-07 from CONTEXT.md):**

| Action | Notification text |
|--------|------------------|
| Member registered | `{displayName} ลงทะเบียนแล้ว (เหลือ {remaining} ที่)` |
| Member cancelled | `{displayName} ยกเลิกแล้ว (เหลือ {remaining} ที่)` |
| Admin removed member | `อัปเดตรายชื่อ (เหลือ {remaining} ที่)` |
| Event is full | `{displayName} ลงทะเบียนแล้ว (เต็มแล้ว)` |

Seats remaining formula: `maxPlayers - registeredCount`. When result is 0: use "เต็มแล้ว" not "เหลือ 0 ที่".
