1. `apps/api/src/webhook/handlers/idempotency.ts` — exact Pattern 3 from RESEARCH.md:
   ```typescript
   import { db, idempotencyKeys } from "@repo/db";

   /**
    * Attempts to insert the webhookEventId. Returns true if this is a new event
    * (should be processed), false if duplicate (skip silently).
    */
   export async function processWithIdempotency(
     webhookEventId: string,
     handler: () => Promise<void>
   ): Promise<void> {
     const result = await db
       .insert(idempotencyKeys)
       .values({ webhookEventId })
       .onConflictDoNothing()
       .returning();

     // result is empty array if conflict (already processed)
     if (result.length === 0) {
       return; // duplicate — skip silently
     }

     await handler();
   }
   ```

2. `apps/api/src/webhook/line.ts` — based on RESEARCH.md Pattern 1. CRITICAL: Do NOT declare a `body` schema on this route. The raw body MUST be read via `request.text()` BEFORE any JSON parsing. The route path is `/webhook/line` (the `/api` prefix is added by the `.group()` in index.ts):
   ```typescript
   import { Elysia } from "elysia";
   import { validateSignature } from "@line/bot-sdk";
   import type { WebhookRequestBody, WebhookEvent } from "@line/bot-sdk";
   import { env } from "../env";
   import { processWithIdempotency } from "./handlers/idempotency";

   async function handleEvent(event: WebhookEvent): Promise<void> {
     await processWithIdempotency(
       event.webhookEventId ?? `${event.type}-${event.timestamp}`,
       async () => {
         // Event dispatch will be implemented in future phases
         // Phase 2: join event handler
         // Phase 4: message event handler
         console.log(`Processing event: ${event.type}`);
       }
     );
   }

   export const lineWebhook = new Elysia()
     .post("/webhook/line", async ({ request, set }) => {
       // MUST call request.text() BEFORE any JSON parsing
       // Do NOT declare body schema on this route — it would consume the stream
       const rawBody = await request.text();
       const signature = request.headers.get("x-line-signature") ?? "";

       if (!validateSignature(rawBody, env.LINE_CHANNEL_SECRET, signature)) {
         set.status = 401;
         return "Unauthorized";
       }

       const payload = JSON.parse(rawBody) as WebhookRequestBody;

       // Return 200 immediately — process events asynchronously
       // Line retries if response is delayed
       void Promise.all(
         (payload.events ?? []).map((event) => handleEvent(event))
       );

       return "OK";
     });
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> /api/webhook/line | Untrusted HTTP POST from Line platform (or attacker) |
| Webhook handler -> Database | Event data written to DB after signature verification |
| Environment -> Process | Secrets loaded from environment variables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | /api/webhook/line | mitigate | `validateSignature()` from `@line/bot-sdk` verifies HMAC-SHA256 with timing-safe comparison; returns 401 on failure; raw body read via `request.text()` before any parsing |
| T-01-02 | Tampering | webhook request body | mitigate | Signature computed against raw bytes; `request.text()` called BEFORE `JSON.parse`; no body schema declared on route to prevent stream consumption |
| T-01-03 | Repudiation | webhook event processing | mitigate | `idempotency_keys` table with `webhookEventId` primary key; `onConflictDoNothing()` prevents duplicate processing |
| T-01-04 | Denial of Service | env.ts startup | mitigate | `@t3-oss/env-core` validates all required env vars at module import time; missing vars crash with descriptive error, not silent runtime failure |
| T-01-05 | Information Disclosure | error responses | mitigate | Webhook returns only "Unauthorized" (401) or "OK" (200); no stack traces or internal details leaked |
</threat_model>
