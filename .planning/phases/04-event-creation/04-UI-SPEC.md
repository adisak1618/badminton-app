## Scope

This phase has two distinct UI surfaces:

1. **LIFF Event Creation Form** вҖ” A single-page scrollable form at `/liff/events/create` inside LINE's mobile webview. Used by admins only.
2. **LINE Flex Message Card** вҖ” A Flex Message bubble posted to the LINE group chat after event creation. Rendered by LINE's native card renderer, not by Next.js.

Design contracts for both surfaces are declared below.

## Visuals

Primary focal point: the "аёӘаёЈа№үаёІаёҮаёӯаёөа№Җаё§аёҷаё—а№Ң" submit button at the bottom of the form вҖ” it is the sole accent-colored (`--primary`) element on the page and anchors the user's eye as the terminal action.

All other form elements (labels, inputs, card container) use neutral surface colors to keep visual weight concentrated on the submit action.

## Interaction States

### LIFF Form Page States

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading (fetching club defaults) | On mount, before fetch resolves | Full-screen centered `<Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />` |
| Auth gate (non-admin or unauthenticated) | LIFF not ready or API returns 403 | `<p className="text-destructive">` with error copy (see Copywriting) |
| Form ready | Club defaults loaded | Render full form with pre-filled fields |
| Submitting | `isSubmitting === true` | Submit button disabled + label changes to "аёҒаёіаёҘаёұаёҮаёӘаёЈа№үаёІаёҮ..." |
| Submit success | API returns 201 | Toast success (sonner) + `liff.closeWindow()` |
| Submit error | API returns 4xx/5xx | Toast error (sonner) вҖ” form stays open, user can retry |
| Field validation error | react-hook-form + zod | `<p className="text-sm text-destructive">` below the field, `aria-describedby` linking input to error |
| Club not linked | API returns 422 (no lineGroupId) | Toast error with specific copy (see Copywriting) |

### Flex Message Card States

Phase 4 only posts the initial card. Card states for Phase 5 (Full, Closed) are out of scope here.

| State | Phase | Spots Display | Register Button |
|-------|-------|--------------|-----------------|
| Open (initial) | Phase 4 | `0/{maxPlayers} аё„аёҷ` in `#22c55e` | Enabled, "аёҘаёҮаё—аё°а№Җаёҡаёөаёўаёҷ" |
| Full | Phase 5 | `{max}/{max} аё„аёҷ` in `#ef4444` | Disabled, "а№Җаё•а№ҮаёЎа№ҒаёҘа№үаё§" |

## Flex Message Card Specification

### Card Structure (LINE Flex Message JSON)

```
bubble
в”ңв”Җв”Җ body (layout: vertical)
в”Ӯ   в”ңв”Җв”Җ text: event.title          вҖ” weight: bold, size: lg
в”Ӯ   в”ңв”Җв”Җ text: formatted date+time  вҖ” size: sm, color: #666666
в”Ӯ   в”ңв”Җв”Җ text: venue name           вҖ” size: sm, action: uri вҶ’ venueMapsUrl (if present)
в”Ӯ   в”ңв”Җв”Җ text: fees                 вҖ” size: sm, color: #666666
в”Ӯ   в”Ӯ     format: "аёҘаё№аёҒаёӮаёҷ {n}аёҝ / аёӘаёҷаёІаёЎ {n}аёҝ"
в”Ӯ   в””в”Җв”Җ text: spots count          вҖ” size: sm, color: #22c55e
в”Ӯ         format: "0/{maxPlayers} аё„аёҷ"
в””в”Җв”Җ footer (layout: horizontal)
    в”ңв”Җв”Җ button: "аёҘаёҮаё—аё°а№Җаёҡаёөаёўаёҷ"        вҖ” style: primary, action: uri вҶ’ LIFF register URL
    в””в”Җв”Җ button: "аёЈаёІаёўаёҘаё°а№Җаёӯаёөаёўаё”"       вҖ” style: secondary, action: uri вҶ’ LIFF details URL
```

### Card Text Formatting Rules

| Data | Format | Example |
|------|--------|---------|
| Event title (auto-generated) | `а№Ғаёҡаё” {day} {shortMonth} - {venueName}` using `Intl.DateTimeFormat("th-TH", { day: "numeric", month: "short" })` (CE year) | `а№Ғаёҡаё” 15 а№ҖаёЎ.аёў. - аёӘаёҷаёІаёЎXYZ` |
| Date + time on card | `Intl.DateTimeFormat("th-TH", { weekday: "short", day: "numeric", month: "short", hour: "2-digit", minute: "2-digit" })` | `аёһ. 15 а№ҖаёЎ.аёў. 18:00` |
| Fees | `аёҘаё№аёҒаёӮаёҷ {shuttlecockFee}аёҝ / аёӘаёҷаёІаёЎ {courtFee}аёҝ` | `аёҘаё№аёҒаёӮаёҷ 50аёҝ / аёӘаёҷаёІаёЎ 200аёҝ` |
| Spots count | `{registered}/{maxPlayers} аё„аёҷ` | `0/16 аё„аёҷ` |

Date formatting approach: CE (Gregorian) with `th-TH` locale вҖ” year omitted from title and card date since Thai badminton groups use CE in daily chat context. Source: RESEARCH.md Architecture Patterns вҖ” Title Auto-Generation (ASSUMED, A1).
