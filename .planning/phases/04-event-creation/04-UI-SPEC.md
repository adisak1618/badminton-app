## Scope

This phase has two distinct UI surfaces:

1. **LIFF Event Creation Form** — A single-page scrollable form at `/liff/events/create` inside LINE's mobile webview. Used by admins only.
2. **LINE Flex Message Card** — A Flex Message bubble posted to the LINE group chat after event creation. Rendered by LINE's native card renderer, not by Next.js.

Design contracts for both surfaces are declared below.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, `space-y-2` between label+input |
| md | 16px | Default element spacing, `space-y-4` between form fields, `px-4` page padding |
| lg | 24px | Section padding, `py-8` page vertical padding, `mb-6` heading bottom margin |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Submit button minimum height: 44px (touch target requirement) — matches `min-h-[44px]` already used in `ProfileForm`
- LIFF page max-width: 384px (`max-w-sm`) — matches established LIFF page pattern

Source: Established patterns from `apps/web/app/liff/profile/page.tsx` and `apps/web/components/liff/profile-form.tsx`

## Form Field Specification

### Event Creation Form Fields (LIFF — `/liff/events/create`)

| Field | Input Type | Required | Default Value | Validation |
|-------|-----------|----------|---------------|------------|
| Event title (ชื่ออีเวนท์) | `<Input type="text">` | No (auto-generated) | Auto: `แบด {day} {month} - {venueName}` | max 255 chars |
| Date + Time (วันที่และเวลา) | `<input type="datetime-local">` | Yes | none | Must be in the future |
| Venue name (ชื่อสนาม) | `<Input type="text">` | Yes | Pre-filled: `clubs.homeCourtLocation` | min 1, max 500 chars |
| Venue Maps URL (Google Maps) | `<Input type="url">` | No | blank (no club default available — see RESEARCH.md Open Q3) | valid URL or empty |
| Shuttlecock fee (ค่าลูก) | `<Input type="number">` | Yes | Pre-filled: `clubs.defaultShuttlecockFee` | min 0, integer |
| Court fee (ค่าสนาม) | `<Input type="number">` | Yes | Pre-filled: `clubs.defaultCourtFee` | min 0, integer |
| Max players (จำนวนสูงสุด) | `<Input type="number">` | Yes | Pre-filled: `clubs.defaultMaxPlayers` | min 1, integer |

Note: Date/time value must have `+07:00` offset appended client-side before POST (see RESEARCH.md Pitfall 5).

## Interaction States

### LIFF Form Page States

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading (fetching club defaults) | On mount, before fetch resolves | Full-screen centered `<Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />` |
| Auth gate (non-admin or unauthenticated) | LIFF not ready or API returns 403 | `<p className="text-destructive">` with error copy (see Copywriting) |
| Form ready | Club defaults loaded | Render full form with pre-filled fields |
| Submitting | `isSubmitting === true` | Submit button disabled + label changes to "กำลังสร้าง..." |
| Submit success | API returns 201 | Toast success (sonner) + `liff.closeWindow()` |
| Submit error | API returns 4xx/5xx | Toast error (sonner) — form stays open, user can retry |
| Field validation error | react-hook-form + zod | `<p className="text-sm text-destructive">` below the field, `aria-describedby` linking input to error |
| Club not linked | API returns 422 (no lineGroupId) | Toast error with specific copy (see Copywriting) |

### Flex Message Card States

Phase 4 only posts the initial card. Card states for Phase 5 (Full, Closed) are out of scope here.

| State | Phase | Spots Display | Register Button |
|-------|-------|--------------|-----------------|
| Open (initial) | Phase 4 | `0/{maxPlayers} คน` in `#22c55e` | Enabled, "ลงทะเบียน" |
| Full | Phase 5 | `{max}/{max} คน` in `#ef4444` | Disabled, "เต็มแล้ว" |
| Closed | Phase 5 | `{n}/{max} คน` | Disabled, "ปิดรับสมัคร" |

## Flex Message Card Specification

### Card Structure (LINE Flex Message JSON)

```
bubble
├── body (layout: vertical)
│   ├── text: event.title          — weight: bold, size: lg
│   ├── text: formatted date+time  — size: sm, color: #666666
│   ├── text: venue name           — size: sm, action: uri → venueMapsUrl (if present)
│   ├── text: fees                 — size: sm, color: #666666
│   │     format: "ลูกขน {n}฿ / สนาม {n}฿"
│   └── text: spots count          — size: sm, color: #22c55e
│         format: "0/{maxPlayers} คน"
└── footer (layout: horizontal)
    ├── button: "ลงทะเบียน"        — style: primary, action: uri → LIFF register URL
    └── button: "รายละเอียด"       — style: secondary, action: uri → LIFF details URL
```

### Card Text Formatting Rules

| Data | Format | Example |
|------|--------|---------|
| Event title (auto-generated) | `แบด {day} {shortMonth} - {venueName}` using `Intl.DateTimeFormat("th-TH", { day: "numeric", month: "short" })` (CE year) | `แบด 15 เม.ย. - สนามXYZ` |
| Date + time on card | `Intl.DateTimeFormat("th-TH", { weekday: "short", day: "numeric", month: "short", hour: "2-digit", minute: "2-digit" })` | `พ. 15 เม.ย. 18:00` |
| Fees | `ลูกขน {shuttlecockFee}฿ / สนาม {courtFee}฿` | `ลูกขน 50฿ / สนาม 200฿` |
| Spots count | `{registered}/{maxPlayers} คน` | `0/16 คน` |

Date formatting approach: CE (Gregorian) with `th-TH` locale — year omitted from title and card date since Thai badminton groups use CE in daily chat context. Source: RESEARCH.md Architecture Patterns — Title Auto-Generation (ASSUMED, A1).

| Field: date/time label | `วันที่และเวลา` | Required |
