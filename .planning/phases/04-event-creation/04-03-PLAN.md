**Step 1: Create Zod validation schema.**

Create `apps/web/lib/validations/event.ts`:

```typescript
import { z } from "zod";

export const eventCreateSchema = z.object({
  title: z.string().max(255).optional(),
  eventDate: z.string().min(1, "กรุณากรอกวันที่และเวลา"),
  venueName: z.string().min(1, "กรุณากรอกชื่อสนาม").max(500),
  venueMapsUrl: z
    .string()
    .url("กรุณากรอก URL ที่ถูกต้อง")
    .max(500)
    .or(z.literal(""))
    .optional(),
  shuttlecockFee: z.coerce.number().int().min(0),
  courtFee: z.coerce.number().int().min(0),
  maxPlayers: z.coerce.number().int().min(1, "กรุณากรอกตัวเลขที่มากกว่า 0"),
});

export type EventCreateFormData = z.infer<typeof eventCreateSchema>;
```

Key details:
- `z.coerce.number()` handles string-to-number conversion from `<input type="number">`
- `venueMapsUrl` allows empty string OR valid URL (optional field)
- Thai error messages match UI-SPEC copywriting contract exactly
- `eventDate` is a string (from `datetime-local` input) — validated as non-empty, timezone handling done in submit logic

The page component:

1. **URL params:** Read `clubId` from `useSearchParams()`. If missing, show error.

2. **State:** `loading` (true initially), `error` (null initially), `defaults` (null initially).

3. **Fetch club defaults on mount:**
   ```
   useEffect when isReady && isLoggedIn:
     fetch(`/api/proxy/events/club-defaults?clubId=${clubId}`)
     if 403 -> setError("คุณไม่มีสิทธิ์สร้างอีเวนท์ กรุณาติดต่อผู้ดูแลสโมสร")
     if !ok -> setError("โหลดข้อมูลไม่ได้")
     else -> setDefaults(data), setLoading(false)
   ```

4. **Form setup with react-hook-form:**
   ```
   const form = useForm<EventCreateFormData>({
     resolver: zodResolver(eventCreateSchema),
     defaultValues: {
       title: "",
       eventDate: "",
       venueName: defaults?.venueName ?? "",
       venueMapsUrl: "",
       shuttlecockFee: defaults?.defaultShuttlecockFee ?? 0,
       courtFee: defaults?.defaultCourtFee ?? 0,
       maxPlayers: defaults?.defaultMaxPlayers ?? 20,
     },
   });
   ```
   NOTE: `defaultValues` must be set AFTER defaults are loaded. Use `form.reset()` when defaults arrive, OR initialize the form inside a conditional render after loading completes.

5. **Submit handler:**
   ```
   async function onSubmit(data: EventCreateFormData):
     // Append Thai timezone offset to datetime-local value (Pitfall 5)
     const eventDateWithTz = data.eventDate + ":00+07:00";

     // Validate date is in the future
     const parsed = new Date(eventDateWithTz);
     if (parsed <= new Date()) {
       form.setError("eventDate", { message: "กรุณาเลือกวันที่ในอนาคต" });
       return;
     }

     const body = {
       clubId,
       title: data.title || undefined,  // omit empty string so API auto-generates
       eventDate: eventDateWithTz,
       venueName: data.venueName,
       venueMapsUrl: data.venueMapsUrl || undefined,
       shuttlecockFee: data.shuttlecockFee,
       courtFee: data.courtFee,
       maxPlayers: data.maxPlayers,
     };

     const res = await fetch("/api/proxy/events", {
       method: "POST",
       headers: { "Content-Type": "application/json" },
       body: JSON.stringify(body),
     });

     if (res.status === 422) {
       toast.error("กลุ่ม LINE ยังไม่ได้เชื่อมกับสโมสร กรุณาเชื่อมก่อนสร้างอีเวนท์");
       return;
     }
     if (!res.ok) {
       toast.error("สร้างอีเวนท์ไม่สำเร็จ กรุณาลองใหม่");
       return;
     }

     toast.success("สร้างอีเวนท์สำเร็จ");
     // Close LIFF window after short delay for toast visibility
     setTimeout(() => {
       liff?.closeWindow();
     }, 1000);
   ```

| T-04-13 | Input Validation | Form fields | mitigate | Client-side zod validation prevents malformed data. Server-side Elysia `t.Object()` schema (Plan 02) is the enforcement boundary. Client validation is UX convenience only. |
