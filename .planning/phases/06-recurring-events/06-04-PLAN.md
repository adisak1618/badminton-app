Create `apps/web/app/liff/events/templates/[id]/edit/page.tsx`:

"use client" page. Layout: `max-w-sm mx-auto px-4 py-8`.

Fetch template by ID: use the list endpoint `GET /api/proxy/event-templates?clubId=${clubId}` and filter client-side with `.find(t => t.id === params.id)`. This avoids adding a new API route — the list is small (typically 1-3 templates per club).

Form with `react-hook-form` + `zodResolver(templateCreateSchema)`:
- Pre-fill all fields from fetched template data
- Same fields as recurring section of create form: venueName, venueMapsUrl, shuttlecockFee, courtFee, maxPlayers, eventDayOfWeek (Select), eventTime (Input type=time), openDayOfWeek (Select), openTime (Input type=time), title
- Submit: PATCH `/api/proxy/event-templates/${params.id}` with form data + `{ clubId }`
- Success: `toast.success("บันทึกการแก้ไขสำเร็จ")`
- Error with maxPlayers conflict: `toast.error("จำนวนผู้เล่นสูงสุดน้อยกว่าผู้ที่ลงทะเบียนอยู่แล้ว")`
- General error: `toast.error("บันทึกไม่สำเร็จ กรุณาลองใหม่")`
- Submit button: `<Button className="w-full min-h-[44px]">บันทึกการแก้ไข</Button>`

Not-admin error state: "คุณไม่มีสิทธิ์จัดการ recurring กรุณาติดต่อผู้ดูแลสโมสร"

Import and use `AlertDialog` from `@repo/ui/components/alert-dialog` for occurrence cancellation. The edit page should also show recent occurrences (events with this templateId). Each occurrence shows a "ยกเลิก" destructive button that opens an AlertDialog:
- Title: "ยกเลิกอีเวนท์นี้?"
- Description: "อีเวนท์จะถูกยกเลิกและแจ้งในกลุ่ม LINE ผู้ลงทะเบียนจะยังถูกเก็บไว้ในระบบ"

- Cancel: `<Button variant="outline">ไม่ยกเลิก</Button>`
- Success: `toast.success("ยกเลิกอีเวนท์สำเร็จ")`

Install AlertDialog if not present: `cd packages/ui && npx shadcn@latest add alert-dialog --yes`
