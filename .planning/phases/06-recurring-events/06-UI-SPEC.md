## Spacing Scale

Declared values (multiples of 4). Derived from existing LIFF page patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`) |
| sm | 8px | Compact element spacing (`space-y-2`, `p-2`) |
| md | 16px | Default element spacing (`space-y-4`, `px-4`) |
| lg | 24px | Section padding (`mb-6`, `pt-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (`py-8` = 32px; `py-16` = 64px) |

Exceptions:
- Touch targets: minimum `min-h-[44px]` on all interactive buttons — matches existing `Button` usage in Phase 5 page (codebase pattern).
- Page container: `max-w-sm mx-auto px-4 py-8` — standard LIFF page wrapper. Carry forward unchanged.

## Color

All values map to CSS custom properties from `packages/ui/src/styles/globals.css` (OKLCH tokens). Do not use raw hex — use semantic token names.

| Role | CSS Token | Usage |
|------|-----------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 — white) | Page background, full-width surfaces |
| Secondary (30%) | `--card` (oklch 1 0 0) + `--muted` (oklch 0.97 0 0) | Cards (`Card` component), section groupings, disabled surfaces |
| Accent (10%) | `--primary` (oklch 0.205 0 0 — near-black) | Primary CTA buttons only: "สร้าง recurring template", "สร้างอีเวนท์ทันที" |
| Destructive | `--destructive` (oklch 0.577 0.245 27.325 — red) | Cancel occurrence button; "ยกเลิก" occurrence confirmation; Full/closed state badge |
| Muted foreground | `--muted-foreground` (oklch 0.556 0 0 — mid-grey) | Placeholder text, secondary metadata, loading spinner |

Accent reserved for:
1. Primary form submit button ("บันทึก recurring template")
2. "สร้างอีเวนท์ทันที" (Create now) button on template detail view

Destructive reserved for:
1. "ยกเลิก occurrence" button
2. Cancellation confirmation dialog confirm action
3. Full/Closed state indicator badge

## Screen Inventory

Three distinct UI surfaces for Phase 6:

### Surface 1: Event Creation Form — Recurring Toggle Extension
**Path:** `apps/web/app/liff/events/create/page.tsx` (extend existing)
**Layout:** Existing `max-w-sm mx-auto px-4 py-8` unchanged.

New fields that appear when recurring toggle is ON (conditional render via `watch("isRecurring")`):

| Field | Component | Label (Thai) | Notes |
|-------|-----------|--------------|-------|
| Recurring toggle | Switch | "สร้างซ้ำทุกสัปดาห์" | Replaces date/time field when ON |
| Event day of week | Select | "วันที่จัดอีเวนท์" | Options: วันจันทร์–วันอาทิตย์ |
| Event time | Input type="time" | "เวลาเริ่ม" | HH:MM, 24h |
| Registration open day | Select | "วันที่เปิดรับสมัคร" | Same options as event day |
| Registration open time | Input type="time" | "เวลาเปิดรับสมัคร" | HH:MM, 24h |

When recurring is ON: hide the `eventDate` datetime-local field. Show the four fields above. All other fields (venueName, fees, maxPlayers, title) remain.

### Surface 2: Template List / Management Page
**Path:** `apps/web/app/liff/events/templates/page.tsx` (new)
**Layout:** `max-w-sm mx-auto px-4 py-8`

Content:
- Page heading: "รายการ recurring" (`text-2xl font-bold mb-6`)
- One `Card` per template showing: title (or auto-generated label), day+time, status Badge
- "สร้างอีเวนท์ทันที" button per template row (variant="outline", full-width, `min-h-[44px]`)
- "แก้ไข" link per template row
- Empty state when no templates exist

### Surface 3: Template Edit Page
**Path:** `apps/web/app/liff/events/templates/[id]/edit/page.tsx` (new)
**Layout:** `max-w-sm mx-auto px-4 py-8`

Content: Same fields as Surface 1 recurring section + base event fields. Pre-filled from existing template. Submit: "บันทึกการแก้ไข" primary Button.

## Interaction Contracts

### Toggle: Recurring Switch (Surface 1)
- Default: OFF (existing one-time event flow unchanged)
- When toggled ON: animate-in of 4 new fields using `transition-all` (tailwind). Datetime-local field fades out (hidden, not removed from DOM — keep registered for non-recurring path).
- Label: "สร้างซ้ำทุกสัปดาห์" with muted helper text below: "ระบบจะสร้างอีเวนท์ทุกสัปดาห์โดยอัตโนมัติ"

### Select: Day of Week
Display order (Thai week, Monday-first):
1. วันจันทร์ (value: 1)
2. วันอังคาร (value: 2)
3. วันพุธ (value: 3)
4. วันพฤหัสบดี (value: 4)
5. วันศุกร์ (value: 5)
6. วันเสาร์ (value: 6)
7. วันอาทิตย์ (value: 0)

### Button: "สร้างอีเวนท์ทันที" (Create Now — D-04)
- Placement: Per-template row on Surface 2, variant="outline"
- Loading state: disable + Loader2 spinner during API call
- Success: toast.success("สร้างและโพสต์อีเวนท์สำเร็จ")
- Error: toast.error("สร้างไม่สำเร็จ กรุณาลองใหม่")

### Occurrence Cancellation (D-12)
- Trigger: "ยกเลิก occurrence" button (destructive variant) on occurrence detail or admin panel
- Confirmation: AlertDialog with:
  - Title: "ยกเลิกอีเวนท์นี้?"
  - Body: "อีเวนท์จะถูกยกเลิกและแจ้งในกลุ่ม LINE ผู้ลงทะเบียนจะยังถูกเก็บไว้ในระบบ"
  - Confirm: "ยกเลิกอีเวนท์" (destructive Button)
  - Cancel: "ไม่ยกเลิก" (outline Button)

### Template Status Badge
| Status | Badge Variant | Thai Label |
|--------|--------------|------------|
| active | default (primary) | "กำลังใช้งาน" |
| paused | secondary | "หยุดชั่วคราว" |
| archived | outline | "เก็บถาวร" |

### Loading States
- Full-page load: centered `<Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />` (matches existing pattern)
- Inline action (submit, create-now): `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` inline in Button
