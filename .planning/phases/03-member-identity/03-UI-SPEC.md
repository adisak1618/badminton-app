## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Form field error message gap, compact element spacing |
| md | 16px | Default element spacing, form field `space-y-4` gap |
| lg | 24px | Section padding, card content padding |
| xl | 32px | Layout gaps between form sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- LIFF pages: full-width, no `max-w-5xl` container. Use `max-w-sm` (384px) centered for profile forms — LIFF runs in LINE in-app browser at narrow mobile width.
- Touch targets for buttons: minimum 44px height (LINE IAB mobile context).

Source: existing layout pattern from `apps/web/app/layout.tsx` (max-w-5xl) overridden for LIFF context per D-03.

Notes:

- Label (14px normal) is used for form field labels (shadcn Label component).

Weights in use: 400 (normal) and 700 (bold) only — maximum 2 weights enforced.

## Screens and Interaction States

### Screen 1: LIFF Loading / Auth Initialization

**Route:** `/liff` (entry point, client component)
**Purpose:** LIFF SDK initializes and auth token is verified. User sees a loading state before being routed to setup or a downstream LIFF page.

| State | Visual |
|-------|--------|
| Loading | Full-screen centered spinner (shadcn Skeleton or a single spinner icon from lucide-react: `Loader2` with `animate-spin`). No text. |
| Error (LIFF not available / token null) | Centered card with heading "Unable to Sign In" and body copy per Copywriting Contract below. |

**Interaction contract:**
- On successful init + auth: server sets session, client is redirected by layout gate.
- On error: show error card. No retry button — user must close and reopen from LINE.

### Screen 2: Profile Setup (`/liff/setup`)

**Route:** `/liff/setup`
**Purpose:** First-time member completes mandatory profile before accessing any other LIFF page.

**Layout:** Full-width mobile page. `max-w-sm mx-auto px-4 py-8`.
**Structure:**
1. Page heading ("Set Up Your Profile") — text-2xl font-bold
2. Subheading copy ("This only takes a moment.") — text-sm text-muted-foreground
3. Card wrapping the form (shadcn Card + CardContent)
4. Three form fields in `space-y-4`:
   - Display Name (Input, pre-filled from LINE ID token `name` claim, required)
   - Skill Level (Select with 4 options: Beginner / Intermediate / Advanced / Competitive, required)
   - Years Playing (Input type="number" min=0, required)
5. Primary CTA button full-width at bottom of form

**Focal point:** Primary visual anchor is the full-width "Set Up Profile" CTA button at the bottom of the card.

**Interaction contract:**
- Display Name: pre-filled on mount from LINE token `name`. Editable.
- Skill Level: shadcn Select. No default selection. Placeholder text: "Select your skill level".
- Years Playing: numeric input, `valueAsNumber: true`. Min 0.
- Submit button: disabled while submitting. Label changes to "Saving..." on in-flight.
- On success: toast "Profile saved" (sonner), then redirect to downstream LIFF page (or `/liff` default).
- Inline validation errors appear below each field in `text-sm text-destructive`.

### Screen 3: Profile Edit (`/liff/profile`)

**Route:** `/liff/profile`
**Purpose:** Existing member updates their profile at any time (MEMB-04).

**Layout:** Same as Setup screen. `max-w-sm mx-auto px-4 py-8`.
**Structure:** Identical to Setup screen except:
- Page heading: "Edit Profile"
- All fields pre-filled with current values from session/API
- CTA label: "Save Changes"

**Interaction contract:**
- Immediate save on tap per D-08. No confirmation dialog.
- On success: toast "Profile updated" (sonner). Stay on page.
- On error: toast "Could not save. Please try again." (sonner, destructive variant).

## Accessibility Notes

- All form fields must have an associated `<label>` (htmlFor + id pair). Use shadcn Label component.
- Select component uses Radix UI Select — keyboard navigable by default.
- Error messages rendered below fields must be linked via `aria-describedby` on the input.
- Loading spinner: add `aria-label="Loading"` to the Loader2 icon wrapper.
- Minimum touch target size: 44px height for Button and Select trigger (LIFF mobile context).
