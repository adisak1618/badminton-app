## Directory Purposes

**apps/**
- Purpose: Production applications built with Next.js App Router
- Contains: Page components, layouts, styles, public assets for user-facing apps
- Key files: `app/page.tsx` (entry point), `app/layout.tsx` (root wrapper), `next.config.js`

**apps/web/**
- Purpose: Primary web application
- Dev server: `next dev --port 3000`
- Dependencies: Next.js 16, React 19, @repo/ui, @repo/eslint-config, @repo/typescript-config

**apps/docs/**
- Purpose: Secondary documentation application (demonstrates monorepo multi-app capability)
- Dev server: `next dev --port 3001`
- Dependencies: Identical to apps/web

**packages/ui/**
- Purpose: Reusable React component library
- Exports: Button, Card, Code components via wildcard export pattern
- No: SSR rendering, Next.js features (pure React components)

**packages/typescript-config/**
- Purpose: Centralized TypeScript compiler options
- Contains: base.json (foundational), nextjs.json (extends base for Next.js), react-library.json (extends base for libraries)
- Usage: Extended via `extends` field in app/package tsconfig.json files

**packages/eslint-config/**
- Purpose: Centralized ESLint rules and plugin configuration
- Contains: base.js (foundation: js, TypeScript, Turbo), next.js (Next.js specific), react-internal.js (library specific)
- Usage: Imported and re-exported in app/package eslint.config.js files

**.planning/codebase/**
- Purpose: GSD analysis documents
- Contains: Architecture and structure documentation for code generation/planning

- `packages/ui/src/code.tsx`: Code/monospace text wrapper

## Naming Conventions

**Files:**
- Components: PascalCase (e.g., `Button`, `Card`, `Code`) exported from `packages/ui/src/`
- Pages: lowercase (e.g., `page.tsx`, `layout.tsx`) following Next.js conventions
- Layouts: `layout.tsx` following Next.js App Router convention
- Modules: camelCase with `.module.css` suffix for scoped styles (e.g., `page.module.css`)
- Config files: camelCase or lowercase with dot notation (e.g., `next.config.js`, `eslint.config.js`)

**Directories:**
- App-level: lowercase (e.g., `web`, `docs`)
- Package-level: kebab-case for scoped names (e.g., `typescript-config`, `eslint-config`)
- Source folders: `src/` convention for packages
- Route groups: Not used in current structure
- Next.js special: `app/` for App Router, `public/` for static assets

**Packages:**
- Workspace scoped packages: `@repo/*` namespace used throughout
- Published scoped packages: `@eslint/js`, `@types/*`, `@next/*` for external dependencies

## Where to Add New Code

**New Feature (Next.js Page/Component):**
- Primary code: `apps/web/app/[feature]/page.tsx`
- Styles: `apps/web/app/[feature]/page.module.css`
- Shared UI: Extract to `packages/ui/src/[ComponentName].tsx`
- Type definitions: Inline in component files or shared TSX files

**New Reusable Component:**
- Implementation: `packages/ui/src/[ComponentName].tsx`
- Export: Automatically available via `@repo/ui/[ComponentName]` (wildcard export)
- Usage in apps: `import { ComponentName } from "@repo/ui/ComponentName"`

**New Configuration:**
- TypeScript: Extend in app's `tsconfig.json` from `@repo/typescript-config/base.json` or `nextjs.json`
- ESLint: Extend in app's `eslint.config.js` from `@repo/eslint-config/base.js` or `next-js.js`

**New Shared Utility:**
- Create in `packages/ui/src/` or new package `packages/[utility-name]/src/`
- Export via package.json wildcard exports or specific exports
- Reference via `@repo/[package-name]/[export]`

**New App:**
- Create in `apps/[app-name]/`
- Copy structure from `apps/web/` (layout.tsx, page.tsx, next.config.js, tsconfig.json, eslint.config.js, package.json)
- Update `package.json` with app-specific name and port in dev script
- pnpm workspace automatically picks up if placed in `apps/` directory

## Special Directories

**apps/*/public/**
- Purpose: Static assets served by Next.js
- Generated: No
- Committed: Yes
- Contains: SVG logos, images

**apps/*/.next/**
- Purpose: Next.js build output and cache
- Generated: Yes (created by `next build`)
- Committed: No (.gitignore)
- Contains: Server code, static pages, build metadata

**packages/ui/dist/**
- Purpose: Built component library output
- Generated: Yes (created by build task if implemented)
- Committed: No (not present in current template)
- Contains: Compiled components

**.turbo/**
- Purpose: Local Turbo task cache
- Generated: Yes
- Committed: No
- Contains: Build artifacts and timing metadata

**node_modules/**
- Purpose: Installed dependencies
- Generated: Yes (by pnpm install)
- Committed: No
- Contains: All transitive dependencies
